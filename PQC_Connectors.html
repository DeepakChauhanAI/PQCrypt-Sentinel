SAST Scanner

Static Analysis Connector

Overview

The SAST scanner is a native static-analysis connector that walks a local source tree and flags cryptographic usage with regex. It lives in backend/app/connectors/sast_connector.py and is invoked through the /api/connectors/sync/sast endpoint or the SAST card on the Connectors page.

How it Works

  1. Discovery

    The target directory is walked with os.walk while pruning excluded directories such as node_modules, .git, __pycache__, .venv, target, build and dist.

  2. Phased Scanning

    Files are scanned in eight phases: Python, Java, Go, JavaScript/TypeScript, dependency manifests, Semgrep (crypto + secrets), container files and Kerberos configuration files.

  3. Findings

    Every regex match becomes a finding containing file, line, category, pattern, code snippet and language.

  4. Persistence

    Creates/updates an Asset, deletes old findings and inserts new findings with risk scores and PQC metadata.

  5. Logging

    A ScanLog entry is written for each phase so progress appears in the Live Execution Console.

AWS PQC Scanner

Cryptographic Discovery

Discovery Scope

When AWS credentials are entered, the AWSPQCScanner scans six AWS services and inventories cryptographic assets for quantum readiness.

ServiceWhat It DiscoversPQC Checks
KMSCustomer managed keys via ListKeys/DescribeKey.RSA/ECC flagged quantum-vulnerable; AES‑256 marked safe.
ACMCertificates via ListCertificates/DescribeCertificate.Signature algorithm, key size and expiry.
ELBv2Load balancers and listeners.TLS policy including PQ-capable policies.
CloudFrontDistributions.Viewer and origin TLS policies.
S3Buckets.SSE-KMS, SSE-S3, SSE-C, DSSE-KMS or no encryption.
IAMServer certificates.Algorithms, key sizes and expiry.

Detailed Service Coverage

  • KMS: Captures KeySpec, key usage and origin. RSA/ECC keys are flagged as quantum-vulnerable while AES-256 symmetric keys are marked safe.
  • ACM: Captures signature algorithm, key size, subject, issuer, SANs and validity window. Classical RSA/ECDSA certificates are flagged vulnerable and near-expiry certificates create separate findings.
  • ELBv2: Captures TLS policies such as ELBSecurityPolicy-TLS13 and ELBSecurityPolicy-PQ variants. Pre‑TLS 1.2 or non-forward-secret policies are flagged.
  • CloudFront: Captures viewer and origin TLS settings and flags weak configurations.
  • S3: Captures bucket default encryption settings and flags buckets without encryption.
  • IAM: Captures uploaded server certificates using the same logic as ACM.
🚫Not Scanned: Secrets Manager secret values, EBS encryption, RDS/DocumentDB encryption-at-rest, Lambda code/env vars, VPC TLS inspection policies and S3 object-level encryption.

AWS KMS Connector

Cloud KMS Discovery

Overview

Enumerates customer-managed AWS KMS keys using boto3. The connector lives in backend/app/connectors/cloud_kms_connector.py.

How it Works

  1. Authenticate

    Resolves credentials from Vault (or direct dict) and instantiates a boto3 KMS client in the chosen region.

  2. List & Describe

    Paginates ListKeys, then DescribeKey for each KeyId, and pulls the default key policy.

  3. Asset Upsert

    Creates or updates an Asset named aws-kms:<key-id> with KeySpec, KeyUsage, Origin and policy stored in metadata.

  4. Findings

    Classifies the key spec (RSA, ECC, HMAC, SM2, AES) and produces PQC findings for quantum-vulnerable keys.

CSV CMDB Connector

Bulk CMDB Import

Overview

Parses a CSV string and upserts Asset records. Lives in backend/app/connectors/csv_connector.py. Useful for seeding the inventory from an existing CMDB export.

Expected CSV Columns

  • name (required, unique per batch)
  • asset_type (server, endpoint, network_device, load_balancer, vpn_gateway, database, web_app, api, container, kubernetes_cluster, cloud_resource, hsm, kms, certificate_authority, smart_card, firmware, saas, other)
  • environment (production, staging, development, testing, unknown)
  • ip_address, fqdn, port (optional)

How it Works

  1. Parse

    Reads CSV with csv.DictReader, normalizes headers to lower-case.

  2. Validate

    Validates asset_type and environment enums, skips duplicate names in the same file.

  3. Upsert

    Creates or updates Asset rows keyed on the name column.

  4. Report

    Returns counts of imported, updated, skipped rows and per-row error messages.

⚠️Limitations: Pure CMDB population; performs no cryptographic analysis. Use SAST or host connectors for crypto detection.

JWT Connector

L4 Application Layer

Overview

Offline + online JWT auditor that decodes header and payload (without signature verification) and classifies the signing algorithm. Lives in backend/app/connectors/jwt_connector.py.

How it Works

  1. Input

    Accepts a list of raw JWT strings (offline) or fetches them from an HTTPS endpoint (online) using Vault-resolved credentials.

  2. Decode

    Decodes header and payload (base64url, no signature check). Extracts alg, kty, key modulus/curve, key size, iss, aud, exp.

  3. Classify

    Maps alg to PQC status via risk_service. ES256/384/512, EdDSA and PS256/384/512 tracked; none, HS1, RS1 are disallowed.

  4. Persist

    Creates an Asset with asset_type=jwt, discovery_source=jwt_audit, and stores the decoded material in metadata. Only the token digest is kept, not the raw token.

⚠️Limitations: Does not verify signatures. JWKS-based key extraction is not currently implemented.

Kubernetes Connector

Container Orchestrator

Overview

Scans a Kubernetes cluster for cryptographic material. Lives in backend/app/connectors/k8s_connector.py and uses the official kubernetes Python client.

Discovery Scope

  • TLS secrets across all namespaces
  • etcd encryption configuration
  • API server certificate
  • Kubelet certificates
  • Certificate Signing Requests (CSRs)

Authentication

  • Kubeconfig YAML (inline in Vault credential)
  • Bearer token + API server host + optional CA cert
⚠️Limitations: Requires cluster credentials and network access to the API server. RBAC must allow reading secrets, CSRs and configmaps.

PKCS#11 HSM Connector

L3 HSM Enumeration

Overview

Enumerates keys and certificates on a PKCS#11 HSM. Lives in backend/app/connectors/pkcs11_connector.py and uses the python-pkcs11 library.

How it Works

  1. Load library

    Loads the vendor PKCS#11 shared library (e.g. /usr/lib/softhsm/libsofthsm2.so).

  2. Authenticate

    Resolves user PIN and optional SO PIN from Vault. Optionally filters by slot ID and token label.

  3. Enumerate

    Lists slots with tokens present, opens each session, and enumerates private/public keys and certificates.

  4. Persist

    Creates Asset rows (asset_type=hsm) and classifies the underlying algorithm.

⚠️Limitations: Requires the HSM hardware or SoftHSM, the vendor PKCS#11 library, and a valid user PIN. Does not extract private key material.

SAML Metadata Connector

Federation Identity

Overview

Parses SAML metadata to extract public keys and certificates. Lives in backend/app/connectors/saml_connector.py.

How it Works

  1. Input

    Accepts a metadata URL (HTTPS) or a raw XML blob. Credentials are resolved from Vault (bearer token) if needed.

  2. SSRF guard

    Validates the URL hostname against the safe-target resolver before fetching.

  3. Parse

    Parses the XML, extracts <ds:X509Certificate> elements and uses the cert_parser to decode them.

  4. Persist

    Creates Assets for the IdP/SP signing certificates and classifies the signature algorithm.

⚠️Limitations: Requires the metadata URL to be reachable from the backend. Signed metadata signatures are not verified.

SSH Agentless Connector

L6 Infrastructure

Overview

Agentless SSH scanner for Linux hosts. Lives in backend/app/connectors/ssh_connector.py and uses paramiko.

Discovery Scope

  • Keystores: JKS, PKCS12, PEM bundles
  • Certificate files on disk
  • OpenSSL version and configuration
  • SSH server / client configuration (weak Kex, MAC, cipher detection)
  • Kerberos configuration

Authentication

  • Username + password
  • Username + private key
  • Optional sudo with separate password (also from Vault)
⚠️Limitations: Requires SSH access (port 22) and a privileged account for full enumeration. sudo is supported but not required.

Oracle TDE Connector

L5 Data Layer

Overview

Scans Oracle Database Transparent Data Encryption (TDE) configuration. Lives in backend/app/connectors/tde_connector.py and uses cx_Oracle.

Discovery Scope

  • V$ENCRYPTION_WALLET status and type
  • V$ENCRYPTED_TABLESPACES for tablespace-level encryption
  • V$DATABASE_KEY_INFO for master key algorithm
  • TDE wallet location on disk

Authentication

  • DB username and password (from Vault)
  • Oracle wallet location (from Vault) for SSL/TDE
⚠️Limitations: Requires Oracle Instant Client, DB user with SELECT_CATALOG_ROLE, and network access to the listener.

WinRM Agentless Connector

L7 Endpoint Layer

Overview

Agentless WinRM scanner for Windows hosts. Lives in backend/app/connectors/winrm_connector.py and uses pywinrm.

Discovery Scope

  • Windows Certificate Store (My, Root, CA, TrustedPublisher, etc.)
  • CNG / NCrypt key storage providers
  • Schannel / CNG registry settings
  • IIS SSL bindings
  • BitLocker status
  • Firmware / UEFI signing information

Authentication

  • NTLM transport (default) over HTTP (5985) or HTTPS (5986)
  • Username + password (from Vault)
⚠️Limitations: Requires WinRM enabled on the target, a privileged domain or local account, and the pywinrm library on the backend host.

Windows Cert Store Connector

L7 Passive Inventory

Overview

Parses certutil -store output to inventory certificates. Lives in backend/app/connectors/winstore_connector.py. Intentionally library-light (stdlib only) so it can run on Linux build agents.

How it Works

  1. Input

    Provide either dump_path (path to a text file) or dump_text (the raw certutil output).

  2. Parse

    Regex-based parser extracts Serial, Issuer, Subject, NotBefore, NotAfter, Cert Hash, Signature algorithm and Public Key.

  3. Persist

    Creates an Asset with asset_type=windows_cert_store for each certificate and classifies the signature algorithm.

Store Parameters

  • store_name: "My", "Root", "CA", "TrustedPublisher", etc.
  • store_kind: "user" or "enterprise" (machine-level)
⚠️Limitations: Requires a separate process to produce the certutil dump. Does not call Win32 APIs directly.

How Findings Are Created

  1. Asset Upsert

    Creates or updates Asset / Certificate records in the database.

  2. Algorithm Classification

    Classifies cryptographic algorithms using the corrected NIST IR 8547 logic and the project PQC algorithm registry.

  3. Algorithm Record

    Creates Algorithm rows and assigns pqc_status (vulnerable, transitioning, hybrid, pqc_ready, safe).

  4. Finding Generation

    Creates findings with severity and risk scores for quantum-vulnerable components.
